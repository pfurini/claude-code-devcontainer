<objective>
Create the plugin library module (`cli/src/lib/plugins.js`) that handles writing `enabledPlugins` and `extraKnownMarketplaces` to `.claude/settings.json`. This follows the direct-write pattern established by `mcp.js` and `settings.js` -- no shell-out to `claude plugin install`. The module provides `applyPresetPlugins` and `removePresetPlugins` functions that the command layer (Plan 03) will consume.
</objective>

  <name>Create plugins.js with applyPresetPlugins</name>
  <files>
    cli/src/lib/plugins.js
  </files>
  <action>
Create `cli/src/lib/plugins.js` with the `applyPresetPlugins(workspaceRoot, presetPlugins, presetMarketplaces = {})` function:

1. Import `readSettings` and `writeSettings` from `./settings.js`.
2. Read current settings via `readSettings(workspaceRoot)`.
3. Merge `extraKnownMarketplaces` from `presetMarketplaces` into `settings.extraKnownMarketplaces` using object spread (additive, does not clobber existing entries). Only process if `presetMarketplaces` has keys. Skip entries for `claude-plugins-official` (always available, needs no registration).
4. Initialize `settings.enabledPlugins` to `{}` if absent (nullish coalescing).
5. For each plugin in `presetPlugins` array (`{ name, marketplace, scope? }`):
   - Construct pluginId as `${name}@${marketplace}`
   - Check if `settings.enabledPlugins[pluginId] === false` (user explicitly disabled). If so, log a warning (`console.warn`) and skip.
   - Otherwise set `settings.enabledPlugins[pluginId] = true`.
6. Write settings via `writeSettings(workspaceRoot, settings)`.
7. Log a message: `"Plugin(s) configured. Restart Claude Code to activate them."` (per risk R1).

Use ES module syntax. Follow the same JSDoc style as `skills.js`.
  </action>

`grep "export async function applyPresetPlugins" cli/src/lib/plugins.js` returns a match. `grep "enabledPlugins" cli/src/lib/plugins.js` returns matches. `grep "readSettings" cli/src/lib/plugins.js` confirms the import.
  </verify>
  <done>
`cli/src/lib/plugins.js` exists with `applyPresetPlugins` exported, handling marketplace registration, user-disabled respect, and enabledPlugins writes.
  </done>

  <action>
Add `removePresetPlugins(workspaceRoot, presetPluginsSnapshot, presetMarketplacesSnapshot = {})` to `cli/src/lib/plugins.js`:

1. Read current settings via `readSettings(workspaceRoot)`.
2. If `settings.enabledPlugins` exists:
   - For each plugin in `presetPluginsSnapshot`, construct pluginId and `delete settings.enabledPlugins[pluginId]`.
   - If `enabledPlugins` is now empty (no keys), delete the key entirely from settings.
3. For `extraKnownMarketplaces` cleanup: use the simpler approach from research R3 -- leave marketplace entries in place on remove. They are harmless and avoiding cross-preset reference checks keeps the code simple. Add a comment explaining this design decision.
4. Write settings via `writeSettings(workspaceRoot, settings)`.

Export the function as a named export.
  </action>
