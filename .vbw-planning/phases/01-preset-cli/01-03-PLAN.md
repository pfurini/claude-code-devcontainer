   i. Ensure gitignore: `ensureGitignore(workspaceRoot)`

  <name>Implement remove command</name>
  <files>
    cli/src/commands/remove.js
  </files>
  <action>
Create `cli/src/commands/remove.js` that exports an async `removeCommand(presetName, options)` function:

1. Resolve workspace root: `process.cwd()`
2. Load state via `loadState(workspaceRoot)`
3. Find the applied entry: `removeApplied(state, presetName)` -- returns entry with snapshot or null
4. If not applied, log error "Preset '<name>' is not currently applied" and exit
5. Using the stored snapshot from the state entry:
   a. Remove settings: `removePresetSettings(workspaceRoot, snapshot.settings)` (if snapshot has settings)
   b. Remove MCP servers: `removePresetMcp(workspaceRoot, snapshot.mcpServers)` (if snapshot has mcpServers)
   c. Remove CLAUDE.md: `removePresetClaudeMd(workspaceRoot, snapshot.claudeMd)` (if snapshot has claudeMd)
   d. Remove skills: `removePresetSkills(workspaceRoot, snapshot.skills)` (if snapshot has skills)
6. Save updated state: `saveState(workspaceRoot, state)`
7. Log success: "Removed preset '<name>'"

The remove command uses the stored snapshot (not the current preset file) to ensure correct reversal even if the preset definition has changed since it was applied.
  </action>
  <verify>
File reads snapshot from state to reverse changes. Calls all remove functions from lib modules. Handles "not applied" case gracefully.
  </verify>
  <done>
`cli/src/commands/remove.js` exports `removeCommand` that uses stored snapshots for clean reversal.
  </done>

  f. `outro('Done!')` -- exit message
- If `isCancel()` returned at any prompt, exit gracefully

**Parse:** `program.parse()` at the end (after the interactive check, so Commander only runs for explicit subcommands)
