  <action>
Edit the Dockerfile to add the `claude-preset` CLI installation. Insert after the Homebrew install line (around line 129, after `brew install ...`) and before the Oh My Zsh block (around line 131):

```dockerfile
# Install claude-preset CLI
COPY --chown=vscode:vscode cli/ /opt/claude-preset/
RUN export PATH="$FNM_DIR:$PATH" && \
  eval "$(fnm env)" && \
  cd /opt/claude-preset && \
  npm install --omit=dev && \
  npm link
```

Key requirements from research:
1. `COPY --chown=vscode:vscode cli/ /opt/claude-preset/` -- follows existing pattern from `COPY --chown=vscode:vscode post_install.py /opt/post_install.py`
2. The RUN step MUST re-initialize fnm with `export PATH="$FNM_DIR:$PATH" && eval "$(fnm env)"` because each RUN layer loses the shell environment. `$FNM_DIR` is already set as a Docker ENV.
3. `npm install --omit=dev` -- skip devDependencies (none in v1, but good practice)
4. `npm link` -- creates global symlink so `claude-preset` is on PATH

Also add `cli/` to `.dockerignore` if a `.dockerignore` exists AND if it would cause issues (check first -- the COPY uses a relative path from build context, so `.dockerignore` excluding `cli/` would break the build). In practice, `cli/` should NOT be in `.dockerignore` since it needs to be copied.
