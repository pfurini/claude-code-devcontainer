<task type="auto">
  <name>Implement merge utility</name>
  <files>
    cli/src/lib/merge.js
  </files>
  <action>
Create `cli/src/lib/merge.js` that exports:

1. `mergeConfig(existing, incoming)` -- deep merges two config objects using `deepmerge` with a custom `arrayMerge` that deduplicates (Set-based union). Used for applying preset settings into existing settings.json.

2. `subtractConfig(current, toRemove)` -- removes entries added by a preset. For arrays: filters out items present in `toRemove`. For objects: recursively subtracts. For scalar keys that match: deletes the key. Used by the `remove` command.

Both functions handle edge cases: null/undefined inputs (treat as `{}`), empty arrays, nested permission objects.

Reference Research R6 for the arrayUnion strategy. Use ES module syntax throughout.
  </action>
  <verify>
File exports both `mergeConfig` and `subtractConfig`. Uses `import deepmerge from 'deepmerge'`. The `arrayMerge` option uses Set-based dedup.
  </verify>
  <done>
`cli/src/lib/merge.js` exports `mergeConfig` and `subtractConfig` with correct deepmerge integration.
  </done>
</task>
<task type="auto">
  <name>Implement state tracking</name>
  <files>
    cli/src/lib/state.js
  </files>
  <action>
Create `cli/src/lib/state.js` that manages `.claude/.preset-state.json` in the workspace root. Exports:

1. `loadState(workspaceRoot)` -- reads and parses state file, returns `{ version: 1, applied: [] }` if missing.
2. `saveState(workspaceRoot, state)` -- writes state file with `JSON.stringify(state, null, 2)`.
3. `isApplied(state, presetName)` -- returns boolean.
4. `addApplied(state, presetName, presetVersion, presetSnapshot)` -- adds entry to `applied` array with `name`, `appliedAt` (ISO timestamp), `presetVersion`, and `snapshot` (the full preset config object, needed for removal).
5. `removeApplied(state, presetName)` -- removes entry from `applied` array, returns the removed entry (with snapshot) or null.
6. `ensureGitignore(workspaceRoot)` -- checks if `.gitignore` contains `.claude/.preset-state.json`, adds it if missing.

The state file path is `path.join(workspaceRoot, '.claude', '.preset-state.json')`.
  </action>
  <verify>
File exports all 6 functions. Uses `fs.readFile`/`fs.writeFile` with proper error handling for missing files. State schema includes `version: 1` and `applied` array with snapshot field.
  </verify>
  <done>
`cli/src/lib/state.js` exports all state management functions with snapshot support for removal.
  </done>

<task type="auto">
  <name>Implement settings and MCP writers</name>
  <files>
    cli/src/lib/settings.js
    cli/src/lib/mcp.js
  </files>
  <action>
Create `cli/src/lib/settings.js`:
1. `readSettings(workspaceRoot)` -- reads `.claude/settings.json` from workspace root, returns `{}` if missing.
2. `writeSettings(workspaceRoot, settings)` -- writes `.claude/settings.json` with 2-space indent + trailing newline.
3. `applyPresetSettings(workspaceRoot, presetSettings)` -- reads current settings, calls `mergeConfig(current, presetSettings)`, writes result. If preset includes `mcpServers`, also sets `enableAllProjectMcpServers: true` in settings.
4. `removePresetSettings(workspaceRoot, presetSettingsSnapshot)` -- reads current, calls `subtractConfig(current, presetSettingsSnapshot)`, writes result.

Create `cli/src/lib/mcp.js`:
1. `readMcpConfig(workspaceRoot)` -- reads `.mcp.json` from workspace root, returns `{ mcpServers: {} }` if missing.
2. `writeMcpConfig(workspaceRoot, config)` -- writes `.mcp.json` with 2-space indent + trailing newline.
3. `applyPresetMcp(workspaceRoot, presetMcpServers)` -- reads current, merges preset's `mcpServers` into `config.mcpServers` (object merge, not deep merge -- server entries are replaced by key). Writes result.
4. `removePresetMcp(workspaceRoot, presetMcpServersSnapshot)` -- reads current, deletes keys present in snapshot from `config.mcpServers`. Writes result (or deletes file if empty).

Both modules import from `./merge.js`. Both handle the case where config files don't exist yet (create them). Write to project scope only (workspace root), never to `~/.claude/`.
  </action>

The `enableAllProjectMcpServers` auto-set is present in applyPresetSettings when MCP servers are involved.
  </verify>

`cli/src/lib/settings.js` and `cli/src/lib/mcp.js` export their respective read/write/apply/remove functions.

Create `cli/src/lib/claudeMd.js`:
1. `applyPresetClaudeMd(workspaceRoot, presetClaudeMd)` -- Given `{ file: "nextjs.md", content: "..." }`:
   a. Creates `.claude/presets/` directory if needed.
   b. Writes content to `.claude/presets/<file>`.
   c. Reads `CLAUDE.md` from workspace root (creates if missing with `# CLAUDE.md\n`).
   d. Checks if `@.claude/presets/<file>` import line already exists; if not, appends it on its own line.
   e. Writes updated CLAUDE.md.
2. `removePresetClaudeMd(workspaceRoot, presetClaudeMd)` -- Given same shape:
   a. Deletes `.claude/presets/<file>` if it exists.
   b. Reads CLAUDE.md, removes the `@.claude/presets/<file>` line, writes back.

Use only project-relative paths in `@` imports (never `@~/...`) per Research risk #4.

Create `cli/src/lib/skills.js`:
1. `applyPresetSkills(workspaceRoot, presetSkills)` -- Given array of `{ repo, skill }`:
   a. For each skill entry, check if `.claude/skills/<skill>/SKILL.md` already exists (idempotent).
   b. If not, run `pnpm dlx skills add <repo> --skill <skill> -y` via `child_process.execSync` or `exec`.
   c. Log success/failure for each skill.
2. `removePresetSkills(workspaceRoot, presetSkills)` -- Given same array:
   a. For each skill, delete `.claude/skills/<skill>/` directory recursively if it exists.

Skills target project scope `.claude/skills/` (not `~/.claude/skills/` which is read-only in the container).
