<task type="auto">
  <name>create-ci-workflow</name>
  <files>
    .github/workflows/ci.yml
  </files>
  <action>
Create `.github/workflows/ci.yml` with the following structure:

```yaml
name: CI
on:
  pull_request:
    branches: [main]
  push:
    branches: [main]
```

**Job 1: lint**
- `runs-on: ubuntu-latest`
- Steps: checkout, install shellcheck (apt-get), run `shellcheck -s bash install.sh`
- ShellCheck targets bash explicitly (install.sh uses bash-specific features)
- If ShellCheck flags SC2034 for unused color variables, the Dev agent should add targeted `# shellcheck disable=SC2034` comments in install.sh only for the color constant block OR add `-e SC2034` to the shellcheck invocation in CI. Prefer the CI flag to minimize install.sh changes.

**Job 2: test**
- `runs-on: ubuntu-latest`
- Steps: checkout, install jq (apt-get), run `bash tests/test_install.sh`
- jq is required for the install.sh functions under test

**Job 3: security**
- `runs-on: ubuntu-latest`
- Steps: checkout, install jq and hadolint, run `hadolint Dockerfile`, run `bash tests/test_security.sh`
- Install hadolint via wget from GitHub releases (latest stable)
- hadolint reads `.hadolint.yaml` automatically from repo root

All jobs run in parallel (no `needs` between them). Each job is independent.
