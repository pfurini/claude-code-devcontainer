Create `tests/test_security.sh` with jq-based security assertions. The script must:
1. Start with `#!/bin/bash` and `set -euo pipefail`
2. Resolve project root as `REPO_ROOT="$(cd "$(dirname "$0")/.." && pwd)"`
3. Define PASS/FAIL counters and a simple assertion helper
4. Run these checks against the actual project files (not fixtures):

**Check 1: SYS_ADMIN absent from runArgs**
```bash
if jq -e '.runArgs[]? | select(test("SYS_ADMIN"))' "$REPO_ROOT/devcontainer.json" >/dev/null 2>&1; then
  echo "FAIL: SYS_ADMIN found in runArgs"; ((FAIL++))
else
  echo "PASS: SYS_ADMIN not in runArgs"; ((PASS++))
fi
```

**Check 2: .devcontainer/ mount is read-only**
Assert that the mounts array contains an entry with both `target=/workspace/.devcontainer` and `readonly`.

**Check 3: Deny rule present in .claude/settings.json**
Assert `.permissions.deny[]` contains `Read(.devcontainer/**)`.

**Check 4: bypassPermissions is intentionally NOT in repo settings**
Assert `.claude/settings.json` does NOT contain `bypassPermissions` (it's set at runtime by post_install.py, not in the repo). This documents the design intent.

5. Print summary and exit 1 if FAIL > 0
6. Make the script executable (`chmod +x`)
  </action>

Create `.hadolint.yaml` at the repo root with targeted rule suppressions:

```yaml
---
# Hadolint configuration for claude-code-devcontainer
# Base images are digest-pinned; apt version pinning is redundant
ignored:
  - DL3008  # apt-get version pinning — upstream images are digest-pinned via sha256
```

Keep DL3016 enabled to catch any future unpinned `npm install -g` additions. The `curl | bash` patterns (Claude Code, fnm, Homebrew) are accepted risks for this project but should remain visible in hadolint output so developers are aware.

Do NOT add DL4006 to ignores — the Dockerfile already has `SHELL ["/bin/bash", "-o", "pipefail", "-c"]` which satisfies it.
